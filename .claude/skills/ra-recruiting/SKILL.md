# RA 채용 프로세스 전체 실행

이 스킬은 BCG RA 채용의 3개 Phase를 순차적으로 실행합니다.
각 Phase 완료 후 사용자에게 다음 단계 진행 여부를 확인합니다.

## 전체 흐름

```
Phase 1 (공고 포스팅) → 진행 확인 → Phase 2 (이력서 스크리닝) → 진행 확인 → Phase 3 (면접 일정 조율)
```

## 실행 규칙

- 각 Phase는 해당 개별 스킬을 Skill 도구로 호출하여 실행합니다.
- Phase 간 전환 시 반드시 AskUserQuestion으로 진행 여부를 확인합니다.
- 사용자가 중단을 선택하면 현재까지의 진행 상황을 요약하고 종료합니다.
- 이전 Phase의 산출물(`data/` 디렉토리)이 다음 Phase의 입력이 됩니다.

## 워크플로우

### Phase 1: 채용 공고 포스팅

1. Skill 도구로 `phase1-ra-job-posting` 스킬을 호출합니다.
2. 스킬이 완료되면 사용자에게 결과를 요약합니다:
   - 프로젝트 설정 저장 여부
   - 포스팅 완료 채널 목록
3. AskUserQuestion으로 다음 단계 진행 여부를 확인합니다:
   - **"Phase 2 (이력서 스크리닝)로 진행"**: 지원 마감일 이후에 진행하는 것을 권장하되, 사용자 판단에 따름
   - **"여기서 중단"**: 현재까지 진행 상황 요약 후 종료

### Phase 2: 이력서 스크리닝

1. Skill 도구로 `phase2-ra-resume-screening` 스킬을 호출합니다.
2. 스킬이 완료되면 사용자에게 결과를 요약합니다:
   - 전체 지원자 수
   - Stage 1/2 통과자 수
   - 최종 선발 인원
3. AskUserQuestion으로 다음 단계 진행 여부를 확인합니다:
   - **"Phase 3 (면접 일정 조율)로 진행"**: 바로 면접 일정 조율 시작
   - **"여기서 중단"**: 현재까지 진행 상황 요약 후 종료

### Phase 3: 면접 일정 조율

1. Skill 도구로 `phase3-ra-interview-setting` 스킬을 호출합니다.
2. 스킬이 완료되면 전체 채용 프로세스 결과를 요약합니다:
   - 공고 포스팅 → 지원자 수 → 선발 인원 → 면접 일정 확정

## 중단 시 처리

사용자가 중간에 중단을 선택한 경우:

1. 현재까지 완료된 Phase와 산출물을 정리하여 보고합니다.
2. 나중에 개별 스킬(`/phase2-ra-resume-screening`, `/phase3-ra-interview-setting`)로 이어서 진행할 수 있음을 안내합니다.
